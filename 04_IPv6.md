## Why IPv6?

**IPv4 address exhaustion** is the primary driver for IPv6 adoption:

| Problem | Detail |
|---------|--------|
| IPv4 address space | ~4.3 billion addresses (2^32) |
| Addresses exhausted | IANA allocated last /8 blocks in 2011 |
| Solution | IPv6 provides 2^128 ≈ 340 undecillion addresses |

### Additional IPv6 Improvements over IPv4

```
Feature               | IPv4                    | IPv6
----------------------|-------------------------|---------------------------
Address size          | 32 bits (4 bytes)       | 128 bits (16 bytes)
Address notation      | Dotted decimal          | Hexadecimal, colon-separated
Header size           | Variable (20-60 bytes)  | Fixed 40 bytes
NAT required?         | Yes (address shortage)  | No (enough addresses)
Fragmentation         | Router & host           | Source host only
Broadcast             | Yes                     | No (uses multicast instead)
ARP                   | Yes (separate protocol) | No (NDP replaces it)
Auto-configuration    | APIPA only (limited)    | SLAAC (full & standard)
IPSec                 | Optional                | Built-in (mandatory in spec)
QoS support           | Limited (DSCP/ToS)      | Flow Label field in header
Header checksum       | Yes                     | No (handled by L4)
```

## IPv6 Address Format

IPv6 addresses are **128 bits** written as **8 groups of 4 hexadecimal digits**, separated by colons.

```
Full format:
2001:0DB8:0000:0001:0000:0000:0000:0001
 ↑     ↑    ↑    ↑    ↑    ↑    ↑    ↑
 G1    G2   G3   G4   G5   G6   G7   G8
(each group = 16 bits = 4 hex digits)
```

### Abbreviation Rules

**Rule 1: Drop leading zeros within each group**
```
2001:0DB8:0000:0001:0000:0000:0000:0001
       ↓
2001:DB8:0:1:0:0:0:1
```

**Rule 2: Replace one consecutive sequence of all-zero groups with `::`**
```
2001:DB8:0:1:0:0:0:1
               ↓
2001:DB8:0:1::1
```

> ⚠️ **Critical Rule:** `::` can only be used **once** in an address. If used twice, it becomes ambiguous (you can't tell how many zeros each `::` represents).

### Abbreviation Examples

| Full Address | Abbreviated |
|-------------|------------|
| 2001:0DB8:0000:0000:0000:0000:0000:0001 | 2001:DB8::1 |
| FE80:0000:0000:0000:020C:29FF:FE00:1234 | FE80::20C:29FF:FE00:1234 |
| 0000:0000:0000:0000:0000:0000:0000:0001 | ::1 |
| 0000:0000:0000:0000:0000:0000:0000:0000 | :: |
| 2001:0DB8:0000:0001:0000:0000:AB00:0001 | 2001:DB8:0:1::AB00:1 |

### Expanding Abbreviated Addresses

**Problem:** Expand `2001:DB8::A1:0:1`  

**Step 1:** Count existing groups → 2001:DB8 (2 groups) + A1:0:1 (3 groups) = 5 groups  
**Step 2:** Missing groups = 8 - 5 = **3 groups**  
**Step 3:** Replace `::` with three groups of `0000`  

```
2001:DB8::A1:0:1
→ 2001:0DB8:0000:0000:0000:00A1:0000:0001
```

## IPv6 Address Types

### 1. Global Unicast Addresses (GUA)

**Equivalent to:** Public IPv4 addresses  
**Range:** `2000::/3` (starts with binary `001` — addresses beginning with 2 or 3)  
**Currently allocated:** `2001::/16` for general use

**Structure:**
```
2001:0DB8:ABCD:0001 : 0000:0000:0000:0001
|←—— 48-bit Global Routing Prefix ——→| |←16-bit Subnet ID→| |←—— 64-bit Interface ID ——→|
  Assigned by ISP/RIR                    Assigned by org          Host portion
```

**Example:** `2001:DB8:1A2B:3::/64` — A company's subnet

---

### 2. Link-Local Addresses (LLA)

**Equivalent to:** APIPA in IPv4 (but used intentionally in IPv6!)  
**Range:** `FE80::/10` (FE80 through FEBF)  
**Prefix used in practice:** Always `FE80::/64`

**Key facts:**
- **Automatically generated** on every IPv6-enabled interface
- **Non-routable** — only valid on the local link (same segment)
- **Required** for IPv6 to function — routers use LLAs as next-hop addresses
- Generated from MAC address using **EUI-64 process** (or randomly)

```
Link-local address example:
FE80::20C:29FF:FE3A:1B2C
```

---

### 3. Loopback Address

**IPv6 Loopback:** `::1`  
**Equivalent to:** 127.0.0.1 in IPv4  
Used to test the local IPv6 stack.

---

### 4. Unspecified Address

**IPv6 Unspecified:** `::`  
**Equivalent to:** 0.0.0.0 in IPv4  
Used as source address before a host has an address assigned.

---

### 5. Unique Local Addresses (ULA)

**Equivalent to:** Private IPv4 addresses (RFC 1918)  
**Range:** `FC00::/7` — in practice `FD00::/8`

**Structure:**
```
FD + 40 random bits + 16-bit Subnet ID + 64-bit Interface ID
```

**Example:** `FD00:1A2B:3C4D::/48`

> 💡 **ULA vs Private IPv4:** Unlike RFC 1918 addresses (which are the same for everyone), ULA includes a **random 40-bit Global ID** to avoid conflicts when organizations merge networks.

---

### 6. Multicast Addresses

**Range:** `FF00::/8`  
**Equivalent to:** Multicast in IPv4 (224.0.0.0/4) — but **no broadcast** exists in IPv6

**Common IPv6 Multicast Addresses:**

| Address | Scope | Meaning |
|---------|-------|---------|
| FF02::1 | Link-local | All nodes (all IPv6 devices on link) |
| FF02::2 | Link-local | All routers |
| FF02::5 | Link-local | All OSPFv3 routers |
| FF02::6 | Link-local | OSPFv3 designated routers |
| FF02::9 | Link-local | All RIPng routers |
| FF02::A | Link-local | All EIGRP routers |
| FF02::1:FF00:0/104 | Link-local | Solicited-node multicast (NDP) |

> ⚠️ **Key Exam Point:** IPv6 has **no broadcast**. Broadcast functionality is replaced by **multicast**. For example, instead of sending to 255.255.255.255, IPv6 sends to FF02::1 (all nodes multicast).

---

### 7. Anycast Addresses

An **anycast** address is assigned to **multiple interfaces** (typically on different devices). Traffic sent to an anycast address is delivered to the **nearest** interface (by routing metric).

**Use cases:**
- DNS servers (same anycast IP on multiple servers worldwide)
- Load balancing
- IPv6 uses the same format as unicast — distinguished by context/configuration

## IPv6 Address Configuration Methods

### Method 1: Manual (Static) Configuration

```cisco
! Cisco IOS
Router(config-if)# ipv6 address 2001:DB8:1:1::1/64
Router(config-if)# ipv6 address FE80::1 link-local
```

### Method 2: EUI-64 (Extended Unique Identifier)

EUI-64 automatically generates the **Interface ID** (last 64 bits) from the device's **MAC address**.

**EUI-64 Process:**
```
MAC Address:   00:1A:2B:3C:4D:5E  (48 bits)
              00:1A:2B : 3C:4D:5E

Step 1: Split MAC in half
         00:1A:2B | 3C:4D:5E

Step 2: Insert FF:FE in the middle
         00:1A:2B:FF:FE:3C:4D:5E

Step 3: Flip 7th bit (Universal/Local bit) of first byte
         00 = 00000000 → flip bit 7 → 00000010 = 02
         02:1A:2B:FF:FE:3C:4D:5E

Step 4: Format as IPv6
         Interface ID: 021A:2BFF:FE3C:4D5E

Result (with prefix 2001:DB8:1:1::/64):
   2001:DB8:1:1:021A:2BFF:FE3C:4D5E
```

**Cisco IOS — Configure EUI-64:**
```cisco
Router(config-if)# ipv6 address 2001:DB8:1:1::/64 eui-64
```

### Method 3: SLAAC (Stateless Address Autoconfiguration)

Hosts configure themselves using information from **Router Advertisement (RA)** messages — **no DHCP server needed**.

```
Host boots up
     ↓
Host sends RS (Router Solicitation) to FF02::2 (all routers)
     ↓
Router responds with RA (Router Advertisement) containing:
  - Prefix: 2001:DB8:1:1::/64
  - Prefix length: /64
  - Default gateway: router's link-local address (FE80::1)
     ↓
Host creates address:
  - Network prefix from RA: 2001:DB8:1:1::
  - Interface ID: generated via EUI-64 or random (RFC 4941 privacy ext.)
  - Full address: 2001:DB8:1:1:021A:2BFF:FE3C:4D5E/64
  - Default GW: FE80::1 (router's link-local)
```

### Method 4: DHCPv6 (Stateful)

Works similarly to DHCPv4 — a DHCPv6 server assigns addresses and provides configuration. The host still uses SLAAC to get the prefix but can also request a full address from DHCPv6.

## ICMPv6 & Neighbor Discovery Protocol (NDP)

**NDP replaces ARP** in IPv6. It uses **ICMPv6 messages** to discover neighbors, routers, and perform address resolution.

### NDP Message Types

| Message | Type Code | Function | IPv4 Equivalent |
|---------|-----------|---------|----------------|
| Router Solicitation (RS) | 133 | Host asks for router info | N/A |
| Router Advertisement (RA) | 134 | Router sends prefix/config info | N/A |
| Neighbor Solicitation (NS) | 135 | "Who has this IP?" (MAC resolution) | ARP Request |
| Neighbor Advertisement (NA) | 136 | "I have this IP, here's my MAC" | ARP Reply |
| Redirect | 137 | Inform host of better route | ICMP Redirect |

### Neighbor Solicitation vs ARP

```
IPv4 ARP:
PC-A → Broadcast (FF:FF:FF:FF:FF:FF): "Who has 192.168.1.20?"

IPv6 NDP (Neighbor Solicitation):
PC-A → Solicited-Node Multicast (FF02::1:FFxx:xxxx): "Who has 2001:DB8::20?"

The solicited-node multicast is derived from last 24 bits of target IP.
Only the target device (or devices sharing those last 24 bits) processes it.
→ Far more efficient than IPv4 broadcast!
```

### Duplicate Address Detection (DAD)

Before using an IPv6 address, a host performs **DAD** to ensure no other device is already using it:

```
Host wants to use: 2001:DB8:1:1::10
Step 1: Host sends NS for 2001:DB8:1:1::10 with source = :: (unspecified)
Step 2: If another device responds with NA → address conflict! Choose another.
Step 3: If no response after waiting → address is unique, proceed to use it.
```

## IPv6 vs IPv4 — Comparison

```
Feature              | IPv4                         | IPv6
---------------------|------------------------------|------------------------------
Address length       | 32 bits                      | 128 bits
Total addresses      | ~4.3 billion                 | ~340 undecillion
Notation             | Dotted decimal (192.168.1.1) | Hex colon (2001:DB8::1)
Loopback             | 127.0.0.1                    | ::1
Private addressing   | RFC 1918                     | ULA (FC00::/7)
Autoconfiguration    | APIPA (limited)              | SLAAC (full, standard)
Header checksum      | Yes                          | No
Broadcast            | Yes                          | No (uses multicast)
ARP                  | Yes                          | No (NDP replaces it)
Fragmentation        | Routers and hosts            | Source host only
Header size          | Variable 20-60 bytes         | Fixed 40 bytes
IPSec                | Optional                     | Required (in spec)
Link-local           | APIPA only when DHCP fails   | Always present (required)
```

## IPv6 Transition Mechanisms

Since not all networks migrated to IPv6 simultaneously, **transition mechanisms** allow IPv4 and IPv6 to coexist.

### 1. Dual-Stack

The device **runs both IPv4 and IPv6 simultaneously** on all interfaces.

```
PC Running Dual-Stack
├── IPv4: 192.168.1.10/24
└── IPv6: 2001:DB8:1::10/64

Can communicate with IPv4-only AND IPv6-only hosts.
```

### 2. Tunneling

**IPv6 packets are encapsulated inside IPv4 packets** to cross IPv4-only networks.

```
IPv6 Network → [IPv6 in IPv4 Tunnel] → IPv4 Network → [IPv6 in IPv4 Tunnel] → IPv6 Network

Common tunnel types:
- 6to4: Automatic tunneling using 2002::/16 prefix
- ISATAP: Intra-Site Automatic Tunnel Addressing Protocol
- GRE tunnels: Manual tunnel configuration on Cisco routers
- 6in4: Static manual tunnels
```

**GRE Tunnel Example (Cisco IOS):**
```cisco
! On Router 1
interface Tunnel0
 ipv6 address 2001:DB8:12::1/64
 tunnel source 203.0.113.1       ! IPv4 source
 tunnel destination 198.51.100.1 ! IPv4 destination
 tunnel mode ipv6ip              ! IPv6 over IPv4
```

### 3. NAT64 / Translation

Allows **IPv6-only hosts to communicate with IPv4-only hosts** by translating between address families. Used by ISPs and large-scale deployments.

## IPv6 on Cisco IOS

### Enable IPv6 Routing

```cisco
! Must enable IPv6 routing (disabled by default)
Router(config)# ipv6 unicast-routing
```

### Configure Interface with IPv6 Address

```cisco
Router(config)# interface GigabitEthernet0/0
Router(config-if)# ipv6 address 2001:DB8:1:1::1/64
Router(config-if)# ipv6 address FE80::1 link-local
Router(config-if)# no shutdown
```

### Verify IPv6 Configuration

```cisco
! Show all IPv6 addresses
Router# show ipv6 interface brief

! Show IPv6 routing table
Router# show ipv6 route

! Show IPv6 neighbors (NDP cache — equivalent of "show arp")
Router# show ipv6 neighbors

! Ping IPv6 address
Router# ping ipv6 2001:DB8:2::1
```

## Key Takeaways

- IPv6 addresses are **128-bit**, written in **hexadecimal** with colon separators
- **Abbreviation rules:** Drop leading zeros; use `::` **once** for consecutive zero groups
- **GUA** (2000::/3) = Public | **ULA** (FC00::/7) = Private | **Link-Local** (FE80::/10) = Non-routable local
- **Link-local addresses** are **mandatory** and auto-generated on every IPv6 interface
- **No broadcast** in IPv6 — replaced by multicast
- **NDP replaces ARP** — uses ICMPv6 Neighbor Solicitation/Advertisement
- **SLAAC** allows hosts to auto-configure without DHCP
- **EUI-64** derives Interface ID from MAC address (splits MAC, inserts FF:FE, flips 7th bit)
- Transition: **Dual-stack** (both protocols), **Tunneling** (IPv6 in IPv4), **NAT64** (translation)
- Enable IPv6 routing on Cisco with: `ipv6 unicast-routing`

## Exam Tips

> 🎯 **High-Frequency Exam Questions:**

1. **"What is the IPv6 loopback address?"** → **::1**
2. **"Which range is used for link-local addresses?"** → **FE80::/10**
3. **"What replaces ARP in IPv6?"** → **NDP (Neighbor Discovery Protocol)**
4. **"What ICMPv6 message type is equivalent to an ARP Request?"** → **Neighbor Solicitation (Type 135)**
5. **"What is SLAAC?"** → **Stateless Address Autoconfiguration** — hosts configure IPv6 addresses using Router Advertisements, no DHCP needed
6. **"How many consecutive `::`s can an IPv6 address have?"** → **Only one**
7. **"What is the abbreviated form of 2001:0DB8:0000:0000:0000:0000:0000:0001?"** → **2001:DB8::1**
8. **"What command enables IPv6 routing on a Cisco router?"** → **`ipv6 unicast-routing`**
